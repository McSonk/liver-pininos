# Comprehensive CSV Data Dictionary

The following tables define every header generated by the `get_volume_summary` function. 

## 1. File Identifiers and Basic Metadata

| CSV Header | Description | Type / Unit | Research Importance |
| :--- | :--- | :--- | :--- |
| `image_path` | Absolute or relative file path to the source CT image. | String | Traceability and debugging of specific outlier cases. |
| `label_path` | Absolute or relative file path to the corresponding segmentation mask. | String | Traceability and debugging of specific outlier cases. |
| `image_shape` | The 3D dimensions of the image array (Depth, Height, Width). | Tuple (voxels) | Indicates the memory footprint and required padding/cropping strategies during preprocessing. |
| `label_shape` | The 3D dimensions of the label mask array. | Tuple (voxels) | Must match `image_shape`. Discrepancies indicate corrupted data pairs. |
| `unique_labels` | A sorted list of all integer class IDs present in the mask. | List[int] | Verifies dataset integrity (e.g., ensuring only classes 0, 1, and 2 are present). |
| `affine_codes` | The anatomical orientation of the volume derived from the NIfTI affine matrix (e.g., 'L', 'A', 'S'). | Tuple[str] | Crucial for standardising orientation (e.g., via `Orientationd`) so the model learns spatial anatomy consistently. |

## 2. Spatial and Geometric Properties

| CSV Header | Description | Type / Unit | Research Importance |
| :--- | :--- | :--- | :--- |
| `spacing_x` | Physical distance between voxels along the X-axis. | mm | Used to calculate physical volumes and standardise isotropic spacing via resampling. |
| `spacing_y` | Physical distance between voxels along the Y-axis. | mm | Used to calculate physical volumes and standardise isotropic spacing via resampling. |
| **`spacing_z`** | **Physical distance between slices along the Z-axis.** | **mm** | **CRITICAL FOR STRATIFICATION.** Highly anisotropic scans (e.g., $> 2.0$ mm) suffer from the "stair-step" artefact and lose crucial 3D contextual information, severely impacting 3D convolutional networks. |
| `voxel_volume_mm3` | The physical volume of a single voxel ($x \times y \times z$). | mm³ | The conversion factor required to translate discrete voxel counts into clinical physical volumes. |

## 3. Slice and Boundary Indices

| CSV Header | Description | Type / Unit | Research Importance |
| :--- | :--- | :--- | :--- |
| `liver_first` | The index of the first axial slice containing liver tissue (class 1). | Integer | Useful for implementing smart cropping transforms (e.g., `CropForegroundd`) to discard empty background slices. |
| `liver_last` | The index of the last axial slice containing liver tissue. | Integer | Useful for implementing smart cropping transforms to reduce memory overhead. |
| `tumor_first` | The index of the first axial slice containing tumour tissue (class 2). | Integer | Helps in designing positive/negative sampling strategies (e.g., `RandCropByPosNegLabeld`). |
| `tumor_last` | The index of the last axial slice containing tumour tissue. | Integer | Helps in designing positive/negative sampling strategies. |

## 4. Volumetric and Ratio Metrics

| CSV Header | Description | Type / Unit | Research Importance |
| :--- | :--- | :--- | :--- |
| `liver_voxels` | Total count of voxels classified as liver (class 1). | Integer | Raw metric for calculating total organ volume. |
| `tumor_voxels` | Total count of voxels classified as tumour (class 2). | Integer | Raw metric for calculating total disease burden. |
| `liver_volume_ml` | Total physical volume of the liver. | mL | Clinical baseline metric. Extreme outliers may indicate segmentation errors in the ground truth or severe hepatomegaly. |
| **`tumour_volume_ml`** | **Total physical volume of all tumours combined.** | **mL** | **CRITICAL FOR STRATIFICATION.** Dictates the difficulty of the case. Small tumours ($< 5$ mL) are prone to false negatives, while massive tumours distort surrounding anatomy. |
| `liver_to_total_ratio` | Fraction of the entire scan volume occupied by the liver. | Dimensionless | Indicates how much of the field-of-view is relevant. Low ratios imply excessive background padding. |
| `tumor_to_total_ratio` | Fraction of the entire scan volume occupied by tumours. | Dimensionless | Highlights extreme class imbalance at the global volume level. |
| `tumor_to_liver_ratio` | Fraction of the liver volume occupied by tumours. | Dimensionless | A clinical proxy for tumour burden relative to organ size. |
| **`has_tumor`** | **Boolean flag indicating if class 2 is present in the volume.** | **Boolean** | **CRITICAL FOR STRATIFICATION.** Ensures the model is trained and evaluated on both healthy livers (negative cases) and pathological livers (positive cases). |

## 5. Liver Intensity (HU) Statistics

*Computed exclusively within the liver mask (class 1).*

| CSV Header | Description | Type / Unit | Research Importance |
| :--- | :--- | :--- | :--- |
| **`liver_hu_mean`** | **Mean Hounsfield Unit of the liver parenchyma.** | **HU** | **CRITICAL FOR STRATIFICATION.** Acts as a robust proxy for the CT contrast phase. Low values ($< 60$ HU) indicate non-contrast or portal phases where tumours are nearly isodense to the liver, drastically increasing segmentation difficulty. |
| `liver_hu_std` | Standard deviation of liver HU. | HU | High standard deviation may indicate heterogeneous liver tissue (e.g., severe steatosis or cirrhosis). |
| `liver_hu_p005` (or) `liver_hu_p01` | 0.5th percentile of liver HU. | HU | Robust minimum, ignoring extreme noise outliers or calcifications. |
| `liver_hu_p995` (or) `liver_hu_p99` | 99.5th percentile of liver HU. | HU | Robust maximum, ignoring extreme noise outliers or contrast pooling. |
| `liver_hu_min` | Absolute minimum HU in the liver mask. | HU | Identifies severe artefacts or fat deposits. |
| `liver_hu_max` | Absolute maximum HU in the liver mask. | HU | Identifies contrast pooling, calcifications, or metallic artefacts. |

## 6. Tumour Intensity (HU) Statistics

*Computed exclusively within the tumour mask (class 2). Returns `None` if no tumour is present.*

| CSV Header | Description | Type / Unit | Research Importance |
| :--- | :--- | :--- | :--- |
| `tumour_hu_mean` | Mean Hounsfield Unit of the tumour. | HU | Indicates the overall radiodensity of the lesion (e.g., hypodense vs. hyperdense). |
| `tumour_hu_std` | Standard deviation of tumour HU. | HU | High values suggest internal heterogeneity, such as necrotic cores or haemorrhage. |
| `tumour_hu_median` | Median Hounsfield Unit of the tumour. | HU | A robust measure of central tendency, less affected by extreme outlier voxels than the mean. |
| `tumour_hu_skewness` | Asymmetry of the tumour HU distribution. | Dimensionless | Positive skew may indicate a mix of viable enhancing tissue and necrotic (low HU) regions. |
| `tumour_hu_p005` (or) `tumour_hu_p01` | 0.5th percentile of tumour HU. | HU | Robust lower bound of tumour intensity. |
| `tumour_hu_p995` (or) `tumour_hu_p99` | 99.5th percentile of tumour HU. | HU | Robust upper bound of tumour intensity. |
| `tumour_hu_min` | Absolute minimum HU in the tumour mask. | HU | Identifies cystic or highly necrotic components within the lesion. |
| `tumour_hu_max` | Absolute maximum HU in the tumour mask. | HU | Identifies calcifications or highly vascularised enhancing regions. |

## 7. Lesion-Level Metrics

*Derived from connected component analysis of the tumour mask.*

| CSV Header | Description | Type / Unit | Research Importance |
| :--- | :--- | :--- | :--- |
| `num_lesions` | Total number of distinct, disconnected tumour masses. | Integer | Multi-focal disease is clinically distinct from solitary masses and presents a harder topological challenge for segmentation models. |
| `lesion_volumes_ml` | List of physical volumes for each individual lesion. | List[mL] | Allows analysis of the size distribution of individual nodules rather than just the aggregate burden. |
| `lesion_equiv_diameters_mm` | List of equivalent spherical diameters for each lesion. | List[mm] | Standard clinical metric for assessing lesion size (e.g., RECIST criteria approximations). |
| `min_lesion_diameter_mm` | Diameter of the smallest detected lesion. | mm | Crucial for evaluating the model's limit of detection (spatial resolution limit). |
| `max_lesion_diameter_mm` | Diameter of the largest detected lesion. | mm | Identifies massive space-occupying lesions that may compress adjacent structures. |
| `mean_lesion_diameter_mm` | Average diameter across all detected lesions. | mm | Provides a general sense of the typical lesion scale in the volume. |

## 8. Texture and Noise Metrics

| CSV Header | Description | Type / Unit | Research Importance |
| :--- | :--- | :--- | :--- |
| `liver_texture_variance` | Variance of HU within the liver mask (excluding tumours). | HU² | High variance in healthy-appearing liver tissue often correlates with underlying parenchymal disease (e.g., fibrosis) or severe imaging artefacts. |
| `liver_noise_estimate` | Robust estimate of image noise. | HU | Calculated as the Interquartile Range (IQR) of liver HU divided by 1.35. High noise levels degrade boundary definition, making gradient-based segmentation more difficult. |

## 9. Global Image Statistics

| CSV Header | Description | Type / Unit | Research Importance |
| :--- | :--- | :--- | :--- |
| `ct_min` | Absolute minimum HU value in the entire 3D image array. | HU | Useful for verifying if the scanner captured the full dynamic range or if aggressive clipping occurred at the source. |
| `ct_max` | Absolute maximum HU value in the entire 3D image array. | HU | Useful for identifying the presence of metallic implants or dense bone which may cause beam-hardening artefacts. |
